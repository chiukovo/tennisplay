# LoveTennis Capacitor 開發待辦 (Backlog)

本文件摘要了 Capacitor 整合中尚未完成或具備高價值的進階功能，供後續開發參考。

## 🛠 核心缺失 (建議最優先處理)
- [ ] **FCM 遷移至 HTTP v1 API**
    - 理由：Legacy API 即將停產，且 v1 支援更多進階通知特性。
    - 需求：需要 Firebase Service Account JSON 對勁。
- [ ] **原生分享整合 (@capacitor/share)**
    - 理由：目前分享功能依賴 Web URL，使用原生分享可直接跳轉手機系統分享介面（如 LINE, Instagram, AirDrop），增加連結點擊率。

## 💎 使用者體驗 (UX) 優化
- [ ] **觸覺回饋 (Haptics)**
    - 理由：在重要動作（如約打成功、傳送訊息）加入微震動，讓 App 感覺更沉浸、更像原生應用。
- [ ] **App 內版本更新偵測**
    - 理由：當我們在 Store 發佈新版本時，能主動提示用戶更新，確保功能同步。
- [ ] **原生大頭貼/圖片裁切**
    - 理由：雖然已有相機整合，但可進一步結合原生裁切插件，讓用戶在上傳前完成完美的頭像縮放。

## 🔐 帳號與安全
- [ ] **整合 Google / Apple 原生快速登入**
    - 理由：除了 LINE 以外，提供更多一鍵登入選項，能最大化手機裝置的轉換率。
- [ ] **權限狀態頁面**
    - 理由：在設定中顯示通知權限、定位權限的狀態，並引導用戶前往手機系統設定開啟。
