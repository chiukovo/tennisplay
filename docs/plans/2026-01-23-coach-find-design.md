# 找教練功能（網頁版）設計草案

## 目標
- 找教練為獨立分頁，但 UI/卡片/篩選與「發現球友」一致。
- 教練在所有球友列表中都要顯示「特殊框線」與教練標示。
- 手機優先：操作流暢、資訊密度適中、CTA 清楚。

## 入口與動線（手機優先）
- 主選單新增「找教練」。
- 「發現球友」頁頂端提供次要入口（卡片或按鈕）。
- 找教練頁：頂部搜尋/篩選 → 卡片列表 → 點擊卡片 → 教練詳情 → 聯絡/預約 CTA。
- 「我是教練」按鈕位於找教練頁頂部（右側或浮動），進入教練資料填寫流程。

## 畫面與資訊架構
### 找教練首頁
- 搜尋與篩選（沿用球友版型）：
  - 關鍵字（教練名/專長）
  - 地區
  - 價格區間
  - 授課方式（個人/團體/線上）
  - 專長 tag
- 教練列表：**沿用球友卡 UI**，新增教練徽章與價格提示。
- 空狀態：提示調整篩選 + 清除篩選。

### 教練卡（沿用球友卡）
- 教練特殊框線（所有列表共用）：`is_coach = true` 必套。
- 教練徽章/標籤 + 價格提示（例如每堂/每小時）。
- 其他資訊沿用球友卡。

### 教練詳情（沿用球友詳情）
- 原本區塊不動，新增「教練專區」區塊：
  - 收費標準（單堂/方案）
  - 上課地點 tag
  - 授課方式
  - 專長/證照（可折疊）
- CTA 改為「聯絡教練／預約」。
- 若教練資料不完整，顯示「洽談為主」文案。

## 我是教練流程
- Step 1：基本資訊（姓名、照片、資歷/證照）
- Step 2：收費標準（單堂/方案）
- Step 3：上課地點 tag + 授課方式
- 預覽 → 送出 → 回到找教練列表

## 狀態與容錯
- Skeleton 與球友頁一致，教練卡 skeleton 也要套框線。
- 若 API 失敗/無網路：顯示空狀態與重試。
- 手機端篩選採收合面板，避免佔畫面。

## 待確認
- 教練欄位完整定義（收費/地點 tag/授課方式/專長/證照）。
- 教練徽章與框線視覺樣式（色系與對比）。
